This function takes a list of integers `nums` and returns the index of any peak element, defined as an element strictly greater than its neighbors (with out-of-bounds neighbors treated as negative infinity). It uses a binary search in O(log n) time: at each step it compares `nums[mid]` with `nums[mid+1]`, narrowing the search to the side that must contain a peak (left if descending, right if ascending), eventually converging on a peak index. It raises a `ValueError` on empty input, and correctly handles single-element lists by immediately returning index 0.
