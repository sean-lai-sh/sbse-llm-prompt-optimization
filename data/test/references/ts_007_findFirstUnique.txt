The `findFirstUnique` function takes a generic array `arr` and returns the first element that appears exactly once in iteration order, or `undefined` if no such element exists (including for an empty array). It uses a two-pass approach: first building a `Map` of element-to-occurrence-count in O(n) time, then re-scanning the array to find the earliest item with count 1, yielding overall O(n) time and O(n) space. Equality is determined by `Map`'s SameValueZero semantics, so reference-typed elements are compared by identity rather than structural equality.
