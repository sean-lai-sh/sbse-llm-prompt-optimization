This function computes the Levenshtein edit distance between two strings `a` and `b`, returning the minimum number of single-character insertions, deletions, or substitutions needed to transform one into the other. It uses the standard dynamic programming recurrence but optimizes memory to O(min-row) by maintaining only the previous and current rows of the DP matrix rather than the full table. It short-circuits on equal strings or when either input is empty, returning 0 or the other string's length respectively.
