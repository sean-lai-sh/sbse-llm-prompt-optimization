Computes the number of distinct combinations of the given `coins` (treated as unlimited supply) that sum to `amount`, returning it as a number. It uses the classic 1D unbounded knapsack DP, iterating coins in the outer loop and amounts in the inner loop to count combinations rather than permutations, with `dp[0] = 1` as the base case. Returns 0 for negative amounts, and naturally returns 1 when `amount` is 0 (the empty combination).
