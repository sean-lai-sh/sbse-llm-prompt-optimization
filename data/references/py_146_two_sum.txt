Solves the classic two-sum problem by scanning the list nums once and using a hash map to record each value's index as it is seen. For each element it computes the complement target - n and returns the (earlier_index, current_index) tuple immediately when the complement already exists in the map, otherwise inserts the current value and continues. Runs in O(n) average time and O(n) space, returns None when no pair sums to target, and reports only the first matching pair found in left-to-right order.
