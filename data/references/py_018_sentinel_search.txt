Performs sentinel linear search by appending the target as a sentinel to the list, scanning forward without a bounds check until it finds the target, then popping the sentinel and verifying that the hit index lies within the original length. It returns the index if the match was a real element or None if it landed on the appended sentinel. The trick removes the per-iteration bounds check to slightly speed up the O(n) scan, but it mutates the input list during execution (appending then popping) and is therefore not safe for read-only or concurrently accessed inputs.
