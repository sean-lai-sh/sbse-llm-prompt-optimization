Finds the single duplicated integer in a list of n+1 numbers drawn from the range 1..n using Floyd's tortoise-and-hare cycle detection on the index-pointer graph implied by the array. It returns the integer at which slow and fast pointers meet after a second walk from the start, running in O(n) time and O(1) extra space without mutating the input. The function assumes the array contains at least one duplicate and valid indices; it will raise IndexError or loop incorrectly on empty input or values outside the implied range.
