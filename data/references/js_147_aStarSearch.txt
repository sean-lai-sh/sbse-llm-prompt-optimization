Performs A* pathfinding on a 2D grid where 0 is passable and 1 is a wall, returning {path, cost, explored} where path is the [row, col] sequence from start to goal or null when unreachable. It supports four-direction or eight-direction (diagonal) movement, picks among Manhattan, Euclidean, and Chebyshev heuristics, and uses gScore/fScore matrices and a cameFrom map for reconstruction. Diagonal moves cost Math.SQRT2 while orthogonal moves cost 1. The open set is implemented as an array re-sorted by fScore on every iteration, giving O(N log N) per pop instead of the O(log N) a true min-heap would provide; explored counts the closed set size at termination.
