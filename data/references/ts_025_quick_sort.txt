Sorts a numeric array in ascending order using in-place quicksort with the Lomuto partition scheme, where the pivot is always the last element of each subrange. It accepts a number[] and returns a new sorted number[] after first shallow-copying the input, giving O(n log n) average time, O(n^2) worst case on already-sorted or reverse-sorted data, and O(log n) average recursion-stack space. The sort is not stable, and recursion depth can degrade to O(n) on adversarial inputs because the pivot choice is fixed.
