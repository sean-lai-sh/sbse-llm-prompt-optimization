Checks whether a string is a syntactically valid HTTP or HTTPS URL using the WHATWG URL parser. It accepts a single argument and returns true only when it is a non-empty string that, after trimming, parses successfully and has a protocol of "http:" or "https:". Non-string inputs, empty/whitespace strings, and parse failures all yield false via a try/catch around the URL constructor. Other schemes such as ftp:, file:, or javascript: are deliberately rejected.
