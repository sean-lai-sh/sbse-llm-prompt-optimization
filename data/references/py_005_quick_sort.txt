Sorts a list of integers using a functional, non-in-place quicksort and returns a new sorted list. It picks the middle element as pivot, partitions the input into three lists of values less than, equal to, and greater than the pivot via list comprehensions, then recursively sorts the outer partitions and concatenates the results. Average-case time is O(n log n) and worst case O(n^2), space is O(n) per recursion level due to the new lists, and the equal-to bucket gracefully handles duplicate keys without quadratic blowup.
