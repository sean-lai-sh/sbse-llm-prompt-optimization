Implements a fixed-capacity LRU cache backed by a Map plus a doubly linked list with sentinel head and tail nodes, then drives it with a list of operations. On get, it moves the accessed node to the front and returns the value, or -1 when missing; on put, it either updates and promotes an existing key or inserts a new node at the front, evicting the tail's predecessor (the least recently used entry) when the Map is at capacity. The dispatcher returns objects describing each "get", "put", and "keys" call, where "keys" walks the list head-to-tail to enumerate keys in MRU order. All get/put operations are O(1); the cache stores keys by Map identity, so it does not deduplicate distinct objects with equal contents.
