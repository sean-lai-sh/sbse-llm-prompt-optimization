Performs an iterative ternary search for target in a sorted array arr by computing two pivots mid1 and mid2 that divide the [low, high] range into thirds, narrowing to the appropriate third based on comparisons against target. Returns the index of a matching element or -1 when target is absent. Runs in O(log3 n) time, which involves more comparisons per step than binary search and is therefore typically slower in practice; uses O(1) auxiliary space and requires arr to be sorted in ascending order.
