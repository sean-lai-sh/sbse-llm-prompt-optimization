Searches a sorted numeric array using ternary search, splitting the active range into three roughly equal parts via mid1 and mid2 and discarding two-thirds on each iteration. It accepts a number[] and a target number and returns the matched index or -1, running in O(log_3 n) time and O(1) space. While asymptotically logarithmic, it performs more comparisons per step than binary search and assumes the array is sorted in ascending order.
