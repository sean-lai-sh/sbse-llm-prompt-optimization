Builds a Huffman code for the input string and returns the encoded bit-string together with the per-character codebook as a (string, dict) tuple. It tabulates character frequencies, then repeatedly pops the two least-frequent subtrees from a heap, prefixing their codes with '0' and '1' and pushing the merged node back, until one tree remains. Empty input returns ("", {}), a single-distinct-character input is special-cased to assign the code "0" so the encoding is non-empty, and the overall cost is O(n + k log k) for a string of length n with k distinct characters.
