Returns a new list that is the input nums rotated to the right by k positions, using the classic three-reversal technique: reverse the entire array, reverse the first k elements, then reverse the remaining n - k elements. The function copies nums first via a slice so the original is not mutated, normalizes k by k % n to handle k larger than length, and returns nums unchanged when n is zero to avoid division by zero. Time complexity is O(n) and space is O(n) for the copy, with the in-place reversals performed by a nested helper using two-pointer index swapping.
