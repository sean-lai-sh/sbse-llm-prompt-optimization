Constructs a Bloom filter sized for the given items and target false-positive rate, then tests each query string for probable membership and returns the results as a list of booleans. Bit-array length m and number of hash functions k are derived from the standard formulas m = ceil(-n ln p / (ln 2)^2) and k = round((m / n) ln 2), with k simulated by hashing "{seed}:{item}" through MD5 for seeds 0..k-1 and reducing modulo m. Membership reports True only when every hash bit is set, so the filter has no false negatives but may produce false positives at approximately the requested rate; an empty items list is handled by clamping n to 1.
