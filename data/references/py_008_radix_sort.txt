Sorts a list of non-negative integers in ascending order using least-significant-digit radix sort with base 10, returning a new list (the local arr is reassigned each pass). For each digit position from ones upward, it distributes numbers into 10 buckets keyed by (num // exp) % 10, then concatenates the buckets to form the next pass's input until the maximum value has no remaining digits. Time complexity is O(d * (n + 10)) where d is the number of digits in the maximum, the empty input is handled by an early return, and negative numbers are not supported because the digit extraction assumes non-negative values.
