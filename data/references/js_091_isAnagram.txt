Tests whether strings a and b are anagrams (case-insensitively) by lowercasing each string, splitting it into a character array, sorting alphabetically, joining back, and comparing the canonical forms for strict equality. Time complexity is O(n log n) due to the sort and O(n) extra space per string. The implementation does not strip whitespace or punctuation, so "dog!" and "god!" match but "dog" and "god " do not, and it does not normalize Unicode (combining marks or different code-point representations of the same grapheme will not match). It treats surrogate pairs as two separate code units because split("") splits on UTF-16 code units, which can corrupt non-BMP characters.
