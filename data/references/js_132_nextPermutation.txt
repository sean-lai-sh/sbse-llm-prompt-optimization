Returns the lexicographically next permutation of a numeric array using the classical in-place algorithm, but on a slice() copy so the original input is not mutated. It scans from the right to find the largest index i with arr[i] < arr[i+1], swaps arr[i] with the rightmost element strictly greater than it, and then reverses the suffix starting at i+1 to make it ascending. When the array is already the largest permutation (i drops to -1), only the suffix reversal runs, which sorts the whole array ascending and effectively wraps to the smallest permutation. Time complexity is O(n) and extra space is O(n) due to the slice copy.
