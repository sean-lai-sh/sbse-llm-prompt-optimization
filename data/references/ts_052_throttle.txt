Returns a throttled version of fn that fires at most once per ms-millisecond window, using a leading-edge invocation when the window has elapsed and a trailing setTimeout to schedule the most recent call when it has not. It tracks lastCall via Date.now() and a pending timer handle, clearing the timer on each invocation so only the latest pending arguments are eventually applied with the original this. The returned function discards fn's return value, has no cancel hook, and the very first call always executes immediately because lastCall starts at 0.
