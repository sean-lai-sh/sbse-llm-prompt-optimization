Escapes HTML-special characters in the input string by chaining six String.prototype.replace calls that substitute &, <, >, ", ', and / with their numeric or named HTML entities. The & replacement runs first to avoid double-escaping the ampersands introduced by later entities. It returns a new string and does not parse HTML structure, so it is suitable for escaping user-supplied text into HTML context but does not strip tags or sanitize attribute payloads.
