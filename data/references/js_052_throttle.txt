Returns a throttled wrapper around fn that ensures it executes at most once per limit milliseconds, using leading-edge invocation when the cooldown has elapsed and a trailing setTimeout to fire the most recent call when still within the window. It tracks lastCall timestamps via Date.now() and stores a single timerId so that intermediate calls during the window only update the queued arguments via closure rather than spawning multiple timers. The wrapper preserves this and arguments through fn.apply, and the trailing call uses the latest args captured by the most recent invocation that scheduled the timer. It does not expose a cancel method, and rapid bursts will execute exactly twice (leading plus trailing) per window.
