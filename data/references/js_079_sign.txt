Returns the sign of a number n as 1 for positive values, -1 for negative values, and 0 for zero (or any value that is neither greater than nor less than 0), using two short-circuiting comparisons in O(1) time and space. Unlike Math.sign, it returns 0 instead of NaN for NaN input because both n > 0 and n < 0 evaluate false for NaN. It also returns 0 for both +0 and -0 rather than preserving the signed zero. Non-numeric inputs follow JavaScript's relational-operator coercion rules.
