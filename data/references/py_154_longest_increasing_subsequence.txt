Returns the length of the longest strictly increasing subsequence of the input integer list using the patience-sorting style tails array. For each element it uses bisect_left to find the leftmost position in tails where the element can extend or replace, appending when it exceeds all tails and overwriting otherwise. The algorithm runs in O(n log n) time with O(n) extra space; an empty list correctly returns 0, and the tails array does not itself reconstruct an actual LIS.
