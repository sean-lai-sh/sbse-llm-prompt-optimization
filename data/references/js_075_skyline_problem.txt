Computes the skyline silhouette of a list of [left, right, height] buildings as an array of [x, height] key points where the maximum active height changes. It generates start events encoded as [left, -height] and end events as [right, height], sorts them lexicographically so that at the same x coordinate taller starts precede shorter starts and starts precede ends, then sweeps left-to-right maintaining a sorted-descending activeHeights array (seeded with 0 for ground level). On each event it inserts or removes the building's height, re-sorting on insertion (O(n log n) per event) and using indexOf+splice on removal, then emits a key point whenever the new max height differs from the previous one. Overall complexity is O(n^2 log n) due to per-event sorting; an empty input returns []. The implementation explicitly notes that a true max-heap would be more efficient.
