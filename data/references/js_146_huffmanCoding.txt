Builds a Huffman code from the character frequencies of the input text and returns {codes, encoded, decode, ratio, freq}, where encoded is the bit string for the input and decode reconstructs text from a bit string by walking the tree. The priority queue is implemented as a sorted array using splice-based insertion, which makes tree construction O(n^2) in the number of distinct characters rather than the heap-based O(n log n). A single-unique-character input is handled as a special case by assigning code "0" and producing an all-zero encoding. Empty or missing input returns empty structures with a no-op decode; the reported ratio assumes 8 bits per original character.
