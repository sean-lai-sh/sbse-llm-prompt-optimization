Creates a debounced wrapper around a function fn that postpones invocation until delay milliseconds have elapsed since the last call, using a single closure-held setTimeout id that is cleared and reset on every call. The returned function preserves this binding via fn.apply and forwards all arguments, but only the most recent invocation actually fires after the quiet period. It also exposes a cancel method that clears any pending timer and resets the id, allowing callers to abort a queued execution. Note that it returns undefined rather than the wrapped function's value since the call is asynchronous.
