Implements a generic LRU cache of given capacity using a Map for O(1) key lookup combined with a doubly linked list bracketed by head and tail sentinels for O(1) recency reordering. get returns undefined for misses and otherwise unlinks the node and re-inserts it just after the head sentinel before returning its value; put updates an existing entry's value and moves it to the front, or for a new key evicts the node before tail (the least-recently-used) when at capacity and inserts a new front node. The size accessor returns the current entry count and all mutating operations preserve the head/tail/Map invariants.
