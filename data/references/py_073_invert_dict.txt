Inverts a dict by swapping keys and values, but handles duplicate values by collecting the original keys into a list under that value. The first time a value is seen the original key is stored as a scalar; on the second collision it is upgraded to a two-element list, and subsequent collisions append. Runs in O(n) time and space; values must be hashable, and the result type is heterogeneous (some values are scalars, some are lists), which can complicate downstream consumers.
