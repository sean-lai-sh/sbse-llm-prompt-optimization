# Genetic Prompt Optimization for Code Summarization

Applies a genetic algorithm to evolve optimal LLM prompt templates for code summarization. Prompt components (role, task description, guard parameters, formatting instructions) are treated as genes — crossover and mutation explore the space of combinations. Fitness blends lexical overlap (ROUGE-L) and semantic similarity (sentence-embedding cosine) against Opus-generated reference summaries, so the search rewards both surface-level accuracy and semantically-equivalent paraphrases.

## Repo Structure

```
/data
  /functions     # 500 self-contained .py / .js / .ts files used as model inputs
  /references    # paired .txt reference summaries generated by Claude Opus
/src             # GA implementation, fitness function, eval loop
/results         # fitness logs, evolved prompt outputs, statistical analysis
```

## Fitness function

```text
fitness(p) = α · ROUGE-L(generated, reference)
           + (1 − α) · cosine_sim(embed(generated), embed(reference))
           − λ_len · length_penalty(prompt)
           − λ_fmt · format_penalty(generated)
```

- `α` (default 0.5): trade-off between lexical (ROUGE-L) and semantic (cosine) similarity
- ROUGE-L rewards token-level overlap with the Opus reference
- Cosine similarity (via `all-MiniLM-L6-v2`) rewards semantically-equivalent rewordings that ROUGE-L would punish
- Length penalty kicks in when the prompt exceeds `MAX_PROMPT_TOKENS`
- Format penalty fires on empty output, code-fenced output, or output >10× reference length

The GA optimizes the blended score directly. Per-metric breakdowns (ROUGE-L only, cosine only, blended) are recorded in `results/` for transparency and ablation.

## How it works

1. GA evolves prompt templates over the `/data/functions` benchmark
2. Each candidate prompt is scored against `/data/references` using the blended fitness above
3. GA is compared against a Random Search baseline (same evaluation budget) using a Wilcoxon rank-sum test (α = 0.05) on best-of-trial fitness, with Cliff's delta for effect size

## Validation

Statistical validation is the GA-vs-RS comparison itself: if GA's best-fitness distribution beats RS's at p < 0.05 with non-negligible effect size, the search is doing meaningful work beyond random sampling. Per-metric breakdowns let us check whether GA's gains come from improved ROUGE-L, improved cosine, or both — useful for the ablation discussion.
